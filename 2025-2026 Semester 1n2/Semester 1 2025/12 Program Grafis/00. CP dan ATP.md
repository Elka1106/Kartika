CP :
Peserta didik mampu menerapkan perintah eksekusi bahasa pemrograman berbasis teks, grafik, dan multimedia yang mengimplementasikan pemrograman terstruktur, pemrograman berorientasi objek lanjutan, pemodelan perangkat lunak, pemrograman antarmuka GUI (Graphical User Interface), dan pemanfaatan pustaka (library).


---

## **Alur Tujuan Pembelajaran (ATP) Kelas 12**
**Mata Pelajaran:** Pemrograman
**Alokasi Waktu:** 32 minggu (2 semester)

---

## **Semester 1: Pemrograman Berorientasi Objek Lanjutan dan Pemodelan Perangkat Lunak**

### **Tujuan Pembelajaran Umum Semester 1**
Peserta didik mampu mengembangkan aplikasi dengan menerapkan pemrograman berorientasi objek lanjutan, pemodelan perangkat lunak, dan integrasi pustaka eksternal.

---

### **Alur Pembelajaran per Minggu (Semester 1)**

| **Minggu** | **Tujuan Pembelajaran Khusus** | **Kegiatan Pembelajaran** | **Penilaian** |
|------------|-------------------------------|---------------------------|---------------|
| **1-2**    | Review OOP dasar dan pengenalan OOP lanjutan | - Review class, object, inheritance <br> - Pengenalan abstract class, interface, polymorphism | Tugas: membuat program dengan abstract class dan interface |
| **3-4**    | Menerapkan pemodelan perangkat lunak | - Pembuatan Use Case Diagram, Class Diagram, Sequence Diagram <br> - Praktik analisis kebutuhan perangkat lunak | Tugas: membuat diagram UML untuk studi kasus |
| **5-6**    | Menerapkan pemrograman GUI lanjutan | - Pengembangan antarmuka grafis dengan library (Tkinter, PyQt, JavaFX) <br> - Praktik membuat form interaktif | Praktik: membuat aplikasi GUI dengan fitur CRUD sederhana |
| **7-8**    | Menerapkan pemrograman multimedia | - Integrasi gambar, suara, dan video menggunakan library (Pillow, Pygame, OpenCV) <br> - Praktik membuat aplikasi dengan elemen multimedia | Proyek mini: aplikasi dengan fitur multimedia |
| **9-10**   | Menerapkan pemanfaatan pustaka eksternal | - Pengenalan dan instalasi library populer (Pandas, NumPy, Requests, dll.) <br> - Praktik menggunakan library untuk memperkaya fungsionalitas | Tugas: program dengan minimal 3 library eksternal |
| **11-12**  | Integrasi OOP, GUI, dan multimedia | - Pengembangan aplikasi sederhana (misal: sistem informasi sekolah, toko online) <br> - Presentasi dan uji coba program | Proyek: aplikasi dengan GUI, OOP, dan multimedia |
| **13-14**  | Pengenalan basis data sederhana | - Pengenalan SQL dan integrasi dengan program (SQLite, MySQL) <br> - Praktik membuat aplikasi dengan basis data | Praktik: aplikasi CRUD dengan basis data |
| **15-16**  | Evaluasi dan proyek akhir semester 1 | - Ujian praktik <br> - Proyek: aplikasi lengkap dengan GUI, OOP, multimedia, dan basis data | Penilaian: ujian praktik dan proyek |

---

## **Semester 2: Pengembangan Aplikasi Berbasis Proyek dan Studi Kasus**

### **Tujuan Pembelajaran Umum Semester 2**
Peserta didik mampu mengembangkan aplikasi berbasis proyek nyata, menerapkan best practice pemrograman, dan mempresentasikan hasil kerja secara profesional.

---

### **Alur Pembelajaran per Minggu (Semester 2)**

| **Minggu** | **Tujuan Pembelajaran Khusus** | **Kegiatan Pembelajaran** | **Penilaian** |
|------------|-------------------------------|---------------------------|---------------|
| **17-18**  | Pengembangan aplikasi berbasis proyek | - Analisis kebutuhan dan perancangan aplikasi <br> - Pembagian tugas kelompok | Tugas: proposal proyek dan rancangan aplikasi |
| **19-20**  | Implementasi proyek | - Pengembangan fitur utama aplikasi <br> - Integrasi semua konsep (OOP, GUI, multimedia, basis data) | Lapor kemajuan: demo fitur utama |
| **21-22**  | Pengujian dan debugging | - Pengujian aplikasi (unit test, integrasi test) <br> - Perbaikan bug dan optimasi kode | Tugas: laporan pengujian dan perbaikan |
| **23-24**  | Dokumentasi dan presentasi | - Pembuatan dokumentasi teknis dan user manual <br> - Persiapan presentasi proyek | Tugas: dokumentasi lengkap |
| **25-26**  | Presentasi proyek akhir | - Presentasi di depan kelas <br> - Demo aplikasi dan tanya jawab | Penilaian: presentasi dan demo aplikasi |
| **27-28**  | Review dan evaluasi | - Evaluasi proyek oleh guru dan teman sekelas <br> - Refleksi pembelajaran selama 3 tahun | Tugas: refleksi tertulis dan perbaikan proyek |
| **29-30**  | Persiapan ujian sekolah | - Review materi selama 3 tahun <br> - Latihan soal ujian praktik | Latihan: ujian simulasi |
| **31-32**  | Ujian sekolah | - Ujian teori dan praktik <br> - Penyerahan portofolio | Penilaian: ujian sekolah dan portofolio |

---

### **Capaian per Fase Pembelajaran**
- **Semester 1:** Peserta didik mampu membuat aplikasi dengan OOP lanjutan, GUI, multimedia, dan integrasi basis data.
- **Semester 2:** Peserta didik mampu mengembangkan aplikasi berbasis proyek nyata, dari analisis hingga presentasi, serta mempersiapkan diri untuk ujian sekolah.

---

### **Penilaian**
- **Penilaian Harian:** Tugas praktik, kuis, partisipasi (30%)
- **Penilaian Tengah Semester:** Ujian praktik dan proyek mini (30%)
- **Penilaian Akhir Semester:** Proyek akhir, presentasi, dan ujian sekolah (40%)

---

### **Catatan:**
- ATP ini dapat disesuaikan dengan bahasa pemrograman yang digunakan (Python, Java, C#, dll.).
- Library dan tools dapat disesuaikan dengan ketersediaan di sekolah.
- Proyek akhir dapat disesuaikan dengan kebutuhan lokal atau industri.
