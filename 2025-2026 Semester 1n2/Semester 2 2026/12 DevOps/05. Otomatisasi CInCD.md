### **2.3. Struktur File Workflow**
File workflow disimpan di folder `.github/workflows/` di repository.
Contoh struktur file:
```yaml
name: Nama Workflow  # Nama workflow (misal: "Node.js CI")

on: [push]  # Event pemicu (misal: push ke repository)

jobs:
  build:  # Nama job
    runs-on: ubuntu-latest  # Mesin yang digunakan
    steps:
      - uses: actions/checkout@v4  # Ambil kode dari repository
      - name: Install Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 18
      - run: npm install  # Instal dependensi
      - run: npm test     # Jalankan test
```

## **3. Contoh Praktis GitHub Actions**

### **Contoh 1: Build & Test Aplikasi Node.js ğŸ“¦**
1. Buat file `.github/workflows/nodejs.yml` di repository:
   ```yaml
   name: Node.js CI

   on: [push]  # Workflow berjalan saat ada push kode

   jobs:
     build:
       runs-on: ubuntu-latest  # Gunakan mesin Ubuntu

       steps:
       - uses: actions/checkout@v4  # Ambil kode dari repository
       - name: Install Node.js
         uses: actions/setup-node@v4
         with:
           node-version: 18
       - run: npm install  # Instal dependensi
       - run: npm test     # Jalankan test
   ```
2. **Hasil:**
   - Setiap kali kamu `git push`, GitHub Actions akan **mengambil kode, instal Node.js, instal dependensi, dan menjalankan test**.

---

### **Contoh 2: Deploy ke GitHub Pages ğŸŒ**
```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]  # Hanya berjalan saat push ke branch main

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v4
    - name: Setup Node.js
      uses: actions/setup-node@v4
      with:
        node-version: 18
    - run: npm install
    - run: npm run build  # Build aplikasi (misal: React/Vue)
    - name: Deploy
      uses: peaceiris/actions-gh-pages@v3
      with:
        github_token: ${{ secrets.GITHUB_TOKEN }}
        publish_dir: ./build  # Folder hasil build
```
- **Hasil:** Aplikasi akan **otomatis di-deploy** ke GitHub Pages setiap kali ada push ke branch `main`.

### **Contoh 3: Build & Push Docker Image ke Docker Hub ğŸ³**
```yaml
name: Docker Build & Push

on:
  push:
    branches: [ main ]

jobs:
  build-and-push:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v4
    - name: Login to Docker Hub
      uses: docker/login-action@v2
      with:
        username: ${{ secrets.DOCKER_HUB_USERNAME }}
        password: ${{ secrets.DOCKER_HUB_TOKEN }}
    - name: Build and push
      uses: docker/build-push-action@v3
      with:
        push: true
        tags: username/repo:latest
```
- **Hasil:** Setiap push ke `main`, GitHub Actions akan **membangun image Docker dan menguploadnya ke Docker Hub**.

### **Contoh 4: Deploy ke Vercel ğŸš€**

## **4. Cara Menyimpan Rahasia (Secrets) ğŸ”**

**Cara menyimpannya:**
