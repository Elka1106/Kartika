## **1. Apa Itu Kontainerisasi?**
**Kontainerisasi** adalah cara mengemas aplikasi beserta semua yang dibutuhkannya (seperti kode, library, dan settingan) ke dalam sebuah **"kontainer"**.

### **Analogi:**
Bayangkan kontainer seperti **kotak makan siang (lunchbox)**.
- Di dalam kotak, ada nasi, lauk, sayur, dan sendok (semua yang dibutuhkan untuk makan).
- Kamu bisa bawa kotak itu ke mana saja (kantor, sekolah, piknik), dan isinya tetap sama.
- Tidak perlu khawatir kalau di tempat tujuan tidak ada nasi atau sendok.

**Dengan kontainerisasi:**
- Aplikasi + semua kebutuhannya = **kontainer**.
- Kontainer ini bisa dipindah-pindah ke mana saja (laptop, server, cloud), dan aplikasi akan berjalan sama persis.

## **2. Docker: Cara Membuat dan Menjalankan Kontainer**

### **2.1. Apa Itu Docker?**
Docker adalah alat untuk membuat, menjalankan, dan mengelola kontainer. Dengan Docker, kamu bisa:
- Membungkus aplikasi ke dalam kontainer.
- Menjalankan kontainer di mana saja.
- Berbagi kontainer dengan orang lain.

### **2.2. Istilah Penting di Docker**

| Istilah         | Penjelasan Sederhana                                                                 | Contoh                                                                 |
|-----------------|--------------------------------------------------------------------------------------|------------------------------------------------------------------------|
| **Image**       | "Resep" atau template untuk membuat kontainer.                                      | Seperti file .exe untuk instal aplikasi.                              |
| **Container**   | "Aplikasi yang sedang berjalan" dari image.                                          | Seperti aplikasi yang sudah dibuka di komputer.                       |
| **Dockerfile** | File teks yang berisi langkah-langkah untuk membuat image.                          | Seperti resep masakan untuk membuat kotak makan siang.                |
| **Docker Hub** | Tempat menyimpan dan berbagi image (seperti Google Drive untuk image Docker).        | Seperti Play Store untuk aplikasi Docker.                              |

---

### **2.3. Instalasi Docker**
1. Download dan instal Docker Desktop: [https://www.docker.com/get-started](https://www.docker.com/get-started)
2. Buka **Docker Desktop** dan pastikan statusnya "Running".

---

### **2.4. Perintah Dasar Docker**

| Perintah                          | Kegunaan                                                                 |
|-----------------------------------|--------------------------------------------------------------------------|
| `docker pull <image>`             | Unduh image dari Docker Hub.                                            |
| `docker run <image>`              | Jalankan kontainer dari image.                                          |
| `docker ps`                       | Lihat kontainer yang sedang berjalan.                                   |
| `docker ps -a`                    | Lihat semua kontainer (termasuk yang sudah berhenti).                   |
| `docker stop <container_id>`      | Hentikan kontainer.                                                     |
| `docker rm <container_id>`        | Hapus kontainer.                                                        |
| `docker images`                   | Lihat semua image yang tersimpan.                                      |
| `docker rmi <image_id>`           | Hapus image.                                                            |
| `docker build -t <nama> .`        | Build image dari Dockerfile.                                            |
| `docker logs <container_id>`      | Lihat log kontainer.                                                    |

---

### **2.5. Contoh Praktis Docker**

#### **Contoh 1: Menjalankan Kontainer NGINX (Web Server)**
```bash
docker run -d -p 80:80 nginx
```
- Buka browser, ketik `http://localhost` untuk melihat halaman NGINX.

#### **Contoh 2: Membuat Image Sendiri (Aplikasi Node.js)**
1. Buat file `Dockerfile`:
   ```dockerfile
   FROM node:18
   WORKDIR /app
   COPY package*.json ./
   RUN npm install
   COPY . .
   EXPOSE 3000
   CMD ["node", "app.js"]
   ```
2. Build image:
   ```bash
   docker build -t my-node-app .
   ```
3. Jalankan kontainer:
   ```bash
   docker run -p 3000:3000 my-node-app
   ```
- Buka browser, ketik `http://localhost:3000` untuk melihat aplikasi.

#### **Contoh 3: Kontainer MySQL**
```bash
docker run --name some-mysql -e MYSQL_ROOT_PASSWORD=my-secret-pw -d -p 3306:3306 mysql
```
- `--name some-mysql`: Beri nama kontainer "some-mysql".
- `-e MYSQL_ROOT_PASSWORD=my-secret-pw`: Set password root MySQL.
- `-d`: Jalankan di background.
- `-p 3306:3306`: Hubungkan port MySQL.

## **3. Kubernetes: Mengatur Banyak Kontainer**

### **3.1. Apa Itu Kubernetes?**
Kubernetes (K8s) adalah alat untuk mengatur banyak kontainer secara otomatis. Misal:
- Jika aplikasi kamu banyak pengunjung, K8s bisa menambah kontainer secara otomatis.
- Jika ada kontainer yang error, K8s bisa menggantinya.

### **3.2. Istilah Penting di Kubernetes**

| Istilah         | Penjelasan Sederhana                                                                 |
|-----------------|--------------------------------------------------------------------------------------|
| **Cluster**     | Kumpulan mesin (node) yang menjalankan kontainer.                                    |
| **Pod**         | Unit terkecil, berisi satu atau lebih kontainer.                                      |
| **Deployment**  | Aturan untuk menjalankan dan memperbarui pod.                                        |
| **Service**     | Cara mengakses pod dari luar.                                                        |
| **Ingress**     | Mengatur akses HTTP/HTTPS dari luar cluster.                                          |

---

### **3.3. Instalasi Kubernetes (Minikube)**
1. Install Minikube: [https://minikube.sigs.k8s.io/docs/start/](https://minikube.sigs.k8s.io/docs/start/)
2. Jalankan cluster lokal:
   ```bash
   minikube start
   ```

---

### **3.4. Perintah Dasar Kubernetes**

| Perintah                          | Kegunaan                                                                 |
|-----------------------------------|--------------------------------------------------------------------------|
| `kubectl get pods`                | Lihat pod yang berjalan.                                                 |
| `kubectl get deployments`         | Lihat deployment.                                                       |
| `kubectl apply -f <file.yaml>`    | Terapkan konfigurasi dari file YAML.                                    |
| `kubectl describe pod <nama>`     | Lihat detail pod.                                                        |
| `kubectl delete pod <nama>`       | Hapus pod.                                                               |
| `kubectl logs <pod>`              | Lihat log pod.                                                           |

---

### **3.5. Contoh Praktis Kubernetes**

#### **Contoh 1: Deployment NGINX**
1. Buat file `nginx-deployment.yaml`:
   ```yaml
   apiVersion: apps/v1
   kind: Deployment
   metadata:
     name: nginx-deployment
   spec:
     replicas: 3
     selector:
       matchLabels:
         app: nginx
     template:
       metadata:
         labels:
           app: nginx
       spec:
         containers:
         - name: nginx
           image: nginx:latest
           ports:
           - containerPort: 80
   ```
2. Terapkan deployment:
   ```bash
   kubectl apply -f nginx-deployment.yaml
   ```
3. Buat service untuk mengakses NGINX:
   ```yaml
   apiVersion: v1
   kind: Service
   metadata:
     name: nginx-service
   spec:
     selector:
       app: nginx
     ports:
       - protocol: TCP
         port: 80
         targetPort: 80
     type: LoadBalancer
   ```
4. Terapkan service:
   ```bash
   kubectl apply -f nginx-service.yaml
   ```
5. Cek service:
   ```bash
   minikube service nginx-service
   ```
- Buka browser, ketik alamat yang muncul untuk melihat halaman NGINX.

## **4. Latihan Praktis**
### **Latihan Docker**
1. Buat Dockerfile untuk aplikasi sederhana (misal: halaman HTML statis).
2. Build image dan jalankan kontainer lokal.
3. Push image ke Docker Hub.

### **Latihan Kubernetes**
1. Buat deployment untuk aplikasi yang sudah dibuat.
2. Buat service untuk mengakses aplikasi dari luar cluster.
3. Lakukan scaling (ubah jumlah replicas).
